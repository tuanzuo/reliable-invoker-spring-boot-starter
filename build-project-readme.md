### 背景

我要使用springboot来实现一个高可靠性调用的组件，命名为`reliable-invoker-spring-boot-starter`，你来帮我实现这个组件，需要按照spring-boot-starter的方式来做；



### 要求

1、使用springboot 使用2.3.12.RELEASE版本

2、使用maven来编译构建项目

3、使用JdbcTemplate来操作表

4、数据库使用mysql



### 功能

1、需要有一个表来记录调用方法的信息；表中需要有一个业务场景字段用来区分不同场景的数据；各个业务可以使用同一个表，也可以单独指定一个表，这些需要通过配置化来实现

2、调用方法之前先把调用方法的信息存入到表中，如果业务执行有事务那么需要把插入表的逻辑放入到业务的事务中去执行；

3、事务执行完成后就执行方法，支持同步执行也支持异步执行，具体选择由使用方来制定

4、需要有一个接口来查询表中未执行成功的数据，进行重试，重试次数和重试的延迟时间需要支持配置，具体次数和延迟时间由使用方来传入

5、需要有一个接口来查询表中几天前已执行成功的数据进行备份，具体几天由使用方来传入，接口的默认实现是删除数据，可以由使用方来覆盖这个默认实现；

6、是否把调用方法的信息存入到表中，要支持配置，具体选择由使用方来制定

7、功能的逻辑，需要定义接口，接口有默认的实现，默认的实现要支持使用方来覆盖

8、对表的操作需要定义接口，接口的默认实现使用JdbcTemplate，默认的实现要支持使用方来覆盖
